A biblioteca seriallib\-RTAI foi concebida para dar funcionalidade de comunica\c{c}\~{a}o serial para processos LINUX com a extens\~{a}o de tempo real RTAI. Ela \'{e} disponibilizada na forma de c\'{o}digo fonte nos arquivos \doxyref{seriallib\-RTAI.c}{p.}{seriallibRTAI_8c} e \doxyref{seriallib\-RTAI.h}{p.}{seriallibRTAI_8h}. Essa biblioteca foi concebida para ser compat\'{\i}vel com processos com multiplos threads, e permite ainda que v\'{a}rios threads acessem a mesma porta serial. No caso, est\~{a}o implementadas fun\c{c}\~{o}es para COM1, COM2, COM3 e COM4. E ainda, a biblioteca implementa fun\c{c}\~{o}es de controle de acesso por sem\'{a}foro, o que permite que uma mesma porta serial possa ser acessada por um s\'{o} thread por vez. Dependendendo do tipo de protocolo, o uso de sem\'{a}foros se faz necess\'{a}rio.

O projeto acompanha um exemplo no diret\'{o}rio test. Para compilar o exemplo, basta fazer make. O resultado \'{e} o arquivo eval\_\-seriallib\-RTAI. Antes de executar esse arquivo \'{e} necess\'{a}rio pelo menos uma vez ap\'{o}s ter iniciado o sistema carregar os m\'{o}dulos do RTAI. Para isso, basta executar o script loadmods.
